# <a name="purpose"></a>目的
* 音声編集用のオープンソースソフト[Audacity](https://www.audacityteam.org/)の使い方を概説する
  * Audacityを[1+SSI+音声再生録音](https://github.com/renesas/rx72n-envision-kit/wiki/1-SSI-%E9%9F%B3%E5%A3%B0%E5%86%8D%E7%94%9F%E9%8C%B2%E9%9F%B3)の動作確認にて使用しているため、動作確認に必要な程度を記す

# <a name="summary"></a>Audacityの概説
## Audacityとは？
* Audacityは音声の再生や編集、録音、音声ファイル書込み/読込みに特化したオープンソースのソフトウェア
* Windowsだけでなく、Mac、Linux上でも動くようだ

## <a name="merit"></a>活用ポイント
* RX72N Envision Kitに実装されているオーディオプロセッサ[D2 Audio](https://www.renesas.com/products/audio-video/audio/digital-sound-processors/device/D2-41051.html)の簡単な動作確認に活用可能
  * テスト用の音声ファイルの作成
  * D2 Audioから出力される音声（サイン波など）が 意図通りかを、Audacityから出力される音声と聞き比べる

# <a name="how_to_use"></a>使用方法
## 準備するもの
* AudacityをインストールしたPC
  * Audacityのインストール方法は割愛

## <a name="sound_creation"></a>音声の作成方法
* [1+SSI+音声再生録音](https://github.com/renesas/rx72n-envision-kit/wiki/1-SSI-%E9%9F%B3%E5%A3%B0%E5%86%8D%E7%94%9F%E9%8C%B2%E9%9F%B3)でも使用したテスト用のサイン波と同等の音声を作成する
  * 左が3.0kHz、右が1.5kHzのサイン波
  * サンプリング周波数が48kHz
  * 量子化ビット数が24ビット

1. Audacityを起動する
   1. もし初期画面で空の音声トラックが作成されている場合は一旦削除する
2. デフォルトのサンプリング周波数を設定する
   1. メニューバーから`編集` - `環境設定`を選択し、設定画面を開く
   2. 左側ペインの`品質`を押す
   3. `サンプリング周波数（デフォルト）`で"48000 Hz"を選択する
      * 画面左下の`プロジェクトのサンプリング周波数（Hz）`からも変更できる
3. デフォルトの量子化ビット数を設定する
   1. メニューバーから`編集` - `環境設定`を選択し、設定画面を開く    
   2. 左側ペインの`品質`を押す
   3. `サンプル形式（デフォルト）`で"24bit"を選択する
   4. ここまで設定できたら`OK`を押し設定画面を閉じる
4. 空の音声トラックを作成する
   1. メニューバーから`トラック` - `新しく追加` - `モノラルトラック`を選択し、空の音声トラックを作成する
   2. 音声トラックの最左列に"モノラル、48000Hz、24bit PCM"と表示されていることを確認する
        * もし表示されていなかったら、上記のデフォルト設定と空のトラック作成をやり直すか、下記の個別設定で変更する
        * 音声トラックの最左列の`音声トラック▼`から名前やサンプリング周波数、量子化ビット数の設定などをトラック別に変更することができる
          * 名前を設定しておくと便利
5. サイン波を生成する（左側の音声用）
   1. 上記で作成した空の音声トラックを選択する（選択すると音声トラックの最左列が青くハイライトされる）
   2. メニューバーから`ジェネレータ―` - `トーン`を選択し、音声作成画面を表示する
   3. `Waveform`で"サイン波"を選択する
   4. `Frequency (Hz)`で"3000"を入力する
   5. `Amplitude (0-1)`で"1"を入力する
      * 相対的な音の大きさの設定であり、お好みで値を決める
   6. `継続時間`で"30秒"を入力する
      * 音声時間の設定であり、お好みで値を決める
        * あまり長い時間を設定すると、音声ファイルに書き出した時にファイルサイズが大きくなるので注意
      * 単位は時間（時分秒）やサンプル数などを選択できる
    7. `OK`を押す
    8. 先ほど選択した音声トラックにサイン波が生成されたことを確認する
       * Ctrl + マウス中ホイールで画面に表示される期間の拡大/縮小ができる
6. サイン波を生成する（右側の音声用）
   1. 4.と5.の操作を繰り返し、別の音声トラックにサイン波を生成する
       * `Frequency (Hz)`には"1500"を入力する
   2. 5.と6.とで作成されたサイン波の周期が異なることを確認する
7. 左側の音声と右側の音声に切り分ける
   1. 左側用として作成した音声トラックを選択する
   2. 音声トラックの最左列で`パン`を"L側に100%"とする
   3. 右側用として作成した音声トラックを選択する
   4. 音声トラックの最左列で`パン`を"R側に100%"とする
8. 生成された音声を再生する
   * 上記までの手順で目的の音声の準備が整っている
   1. 画面上の`再生`ボタンを押す
   2. 目的の高さで音声が再生されることを確認する

## <a name="sound_mix"></a>左側用と右側用の音声の合成方法
* [音声の作成方法](#sound_creation)で作成した左側用と右側用の音声を合成する
1. 左側用の音声トラックを選択する
2. 音声トラックの最左列で`ミュート`を選択する
3. 右側用の音声トラックを選択する
4. 音声トラックの最左列で`ミュート`を選択する
5. Ctrl + クリックで左側用の音声トラックと右側用の音声トラックの両方を選択状態にする
  * 左側用と右側用の音声トラックの最左列が青くハイライトされたかどうかで判別可能
6. メニューバーから`トラック` - `ミックス` - `新しいトラックにミックスして作成`を選択する
7. 新しく音声トラックが作成されるので、音声トラックの最左列が"ステレオ、48000Hz、24bit PCM"と表示されていることを確認する
   * モノラルではなくステレオという点に注意
   * 左側用と右側用の音声トラックはそれぞれ`パン`でどちらか一方に偏らせているので、ステレオのチャネルそれぞれにきれいなサイン波が維持されている
8. 生成された音声を再生する
   1. 左側用と右側用の音声トラックについて、音声トラックの最左列でぞれぞれ`ミュート`を選択する
   2. 合成した音声トラックの方は`ソロ`となっていることを確認する
   3. 画面上の`再生`ボタンを押す
   4. 目的の高さで音声が再生されることを確認する（[音声の作成方法](#sound_creation)で再生した音と変わらないはず）

## <a name="sound_file_creation"></a>音声ファイルの作成方法
* [左側用と右側用の音声の合成方法](#sound_mix)で作成した音声をWAVファイルとして書き出す
1. 合成した音声トラック以外は`ミュート`になっていることを確認する（合成した音声トラックが`ソロ`になっていることを確認する）
2. メニューバーから`ファイル` - `書き出し` - `WAVとして書き出し`を選択する
3. WAVファイルを保存する場所とファイル名を指定する
4. `エンコーディング`を任意に指定し`保存`を押す
5. メタデータタグを編集画面で任意に値を入力し（何も入力しなくてもOK）、`OK`を押す
6. WAVファイルが作成されたことを確認する
